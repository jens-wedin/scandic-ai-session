# Prompt Starters for Scandic Design Team

Ready-to-use prompts for common tasks. Copy, paste, and customize.

---

## Design Review

**Quick component review:**
> Review this [component name] design. Check visual hierarchy, spacing consistency, design system alignment, and accessibility. Flag any issues.

**Full page review:**
> I'm designing [page/feature name] for [context: mobile web / desktop / responsive]. Review the layout, hierarchy, and flow. Check that it follows Scandic's design principles and is accessible. Here's what it should accomplish: [user goal].

**Figma-to-code comparison:**
> Compare this Figma component to its Storybook implementation. Check for: token usage, variant parity, spacing accuracy, and missing states. Flag any drift.

---

## Research

**Interview synthesis:**
> I conducted [number] user interviews about [topic]. Here are my notes: [paste notes]. Synthesize into: key themes, notable quotes, and design implications. Highlight any surprising findings.

**Competitive analysis:**
> Analyze how [competitor names] handle [feature/flow]. Compare to Scandic's current approach. Identify: what they do better, what we do better, and opportunities to differentiate.

**Research planning:**
> I need to research [topic/question] for [project]. Help me plan: research questions, recommended method, participant criteria, and a discussion guide outline.

**Journey mapping:**
> Map the user journey for [scenario, e.g., "booking a room as a wheelchair user"]. Include: touchpoints, user emotions, pain points, and opportunities. Consider both digital and physical touchpoints.

## Accessibility

**Quick a11y check:**
> Check this [component/page/flow] for WCAG 2.2 AA compliance. Focus on: contrast, keyboard access, screen reader support, and touch targets.

**Accessibility annotation:**
> Help me create accessibility annotations for this design. I need: focus order, ARIA roles and labels, alt text suggestions, and any semantic HTML recommendations.

**Contrast check:**
> I'm using these colors: foreground [color] on background [color]. Check the contrast ratio and tell me if it passes WCAG 2.2 AA for normal text, large text, and UI components. Suggest alternatives if it fails.

**Inclusive design review:**
> Review this [feature/flow] through an inclusive design lens. Consider users with: visual impairments, motor disabilities, cognitive disabilities, and situational limitations (e.g., one-handed use, bright sunlight, noisy environment).

---

## Design System

**Token audit:**
> Audit this component for design token usage. List any hardcoded values that should be tokens, and suggest token names following our naming convention: [category]-[property]-[variant]-[state].

**New component proposal:**
> I need a new [component name] component. Help me define: purpose, variants (size, style, state), props API, token requirements, accessibility requirements, and usage guidelines.

**Component documentation:**
> Write usage documentation for [component name]. Include: when to use it, when NOT to use it, required vs optional props, accessibility considerations, and do's/don'ts with examples.

---

## General Design

**Design critique:**
> I'm stuck on [design problem]. Here's my current approach: [describe]. Give me honest critique and 3 alternative approaches to consider. Keep Scandic's design principles in mind.

**Copy review:**
> Review this UI copy for: clarity, tone (warm, helpful, Scandic-appropriate), inclusivity, and brevity. Suggest improvements. Context: [where this copy appears].

**Responsive strategy:**
> How should [component/layout] adapt across breakpoints? Consider: mobile (320px), tablet (768px), desktop (1024px), and wide (1440px). Focus on content priority and interaction changes.
